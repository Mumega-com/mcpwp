# WordPress.org Plugin Assets

This directory contains assets for the WordPress.org plugin directory listing.

## Required Files

These files should be created before WordPress.org submission:

### Banners
- `banner-772x250.png` - Low resolution banner (required)
- `banner-1544x500.png` - High resolution banner (retina)

### Icons
- `icon-128x128.png` - Low resolution icon (required)
- `icon-256x256.png` - High resolution icon (retina)
- `icon.svg` - SVG icon (optional, best quality)

### Screenshots
Screenshots should be numbered and match the readme.txt descriptions:
- `screenshot-1.png` - Admin settings page
- `screenshot-2.png` - API key management
- `screenshot-3.png` - REST API documentation

## Specifications

### Banners
- Dimensions: 772x250 (1x) or 1544x500 (2x)
- Format: PNG or JPG
- Keep important content in center (cropping may occur on mobile)

### Icons
- Dimensions: 128x128 (1x) or 256x256 (2x)
- Format: PNG with transparency recommended
- Simple, recognizable design

### Screenshots
- Any reasonable dimensions (will be resized)
- PNG format recommended
- Number matches readme.txt screenshot descriptions

## Branding Guidelines

Canonical brand guidance now lives in `docs/BRAND_KIT.md`.

Current shipped identity:

- **Name:** MCPWP
- **Primary descriptor:** AI operations for WordPress through MCP
- **Short tagline:** Control WordPress with AI
- **Core colors:** `#0B1220`, `#1B4DFF`, `#FFFFFF`
- **Primary motif:** Compass / navigation mark with subtle technical grid

## Design Suggestions

The banner should communicate:
1. Controlled WordPress operations through MCP
2. Technical trust and production readiness
3. Direction, navigation, and structured workflows

Icon suggestions:
- Compass / pilot directionality
- AI circuit hints used sparingly
- High contrast geometry that reads clearly at small sizes
