# MCPWP Brand Kit

This document is the canonical brand reference for MCPWP across the website, plugin assets, docs, product UI, demos, and partner materials.

## Brand Core

**Brand name:** MCPWP

**Category:** WordPress MCP platform

**Primary descriptor:** AI operations for WordPress through MCP

**Short tagline:** Control WordPress with AI

**One-line pitch:** MCPWP turns a WordPress site into an MCP server so AI agents can safely manage content, design, commerce, and admin workflows.

**Primary audiences:**
- WordPress agencies
- Freelance builders
- WooCommerce operators
- AI automation developers
- Technical site owners

## Positioning

### What MCPWP is

MCPWP is the control layer between AI clients and self-hosted WordPress. It exposes structured tools, role-scoped access, and site-aware workflows so assistants can operate WordPress safely.

### What MCPWP is not

- Not a generic chatbot plugin
- Not a no-code page builder
- Not a black-box autonomous agent
- Not only for content generation

### Core value pillars

1. **Control**  
   Users can operate WordPress with precise, structured tools instead of brittle browser automation.

2. **Safety**  
   Role-scoped API keys, tool boundaries, and auditability matter as much as raw automation.

3. **Speed**  
   Repetitive work becomes promptable and repeatable.

4. **WordPress depth**  
   MCPWP is built for real WordPress operations: Elementor, WooCommerce, media, SEO, menus, and admin workflows.

5. **Operator credibility**  
   The tone should feel technical, reliable, and production-ready.

## Messaging Hierarchy

Use this order when writing homepage copy, deck titles, ads, or demo intros:

1. WordPress control through MCP
2. Safe AI operations
3. Built for agencies and builders
4. Real tool coverage across content, design, commerce, and admin

### Approved short descriptions

- AI operations for WordPress through MCP
- Safe WordPress control for AI agents
- The MCP layer for self-hosted WordPress
- Structured WordPress tools for Claude, Cursor, Codex, and other AI clients

### Avoid

- "Magic website builder"
- "Fully autonomous WordPress employee"
- "No setup needed"
- "Replaces your team"
- "Unlimited AI WordPress automation" unless technically true

### Mark meaning

The compass mark represents navigation, control, direction, and guided operation. The upward pointer suggests forward motion and operator intent rather than autonomous drift.

### Logo rules

- Prefer the full lockup `MCPWP` plus descriptor in marketing placements.
- Use the compass icon alone for app icons, favicons, plugin cards, and avatars.
- Use the full-color gradient version on dark or neutral backgrounds.
- Use all-white when placing the mark on electric blue or image backgrounds.
- Do not rotate, stretch, outline, or add drop shadows to the mark.
- Do not place the logo on noisy screenshots without a dark overlay.

### Clear space

Keep at least the width of the center dot around the icon on all sides. For the wordmark, keep clear space equal to the height of the `M`.

### Minimum sizes

- Icon: `24px`
- Wordmark: `120px`
- Banner lockup: scale proportionally, never below legible subtitle size

## Color System

The shipped visual identity already centers on deep navy, electric blue, and white. This kit extends that palette into a full UI and marketing system without changing the brand's core look.

|---|---|---|
| `Ink 950` | `#0B1220` | Main background, footer, dark surfaces |
| `Blue 600` | `#1B4DFF` | Primary brand accent, CTA, links |

- Primary gradient: `linear-gradient(135deg, #0B1220 0%, #1B4DFF 50%, #0B1220 100%)`
- Accent gradient: `linear-gradient(135deg, #1B4DFF 0%, #5B7CFF 100%)`

### Usage rules

- Use `Ink 950` as the dominant anchor color.
- Use `Blue 600` sparingly to direct focus: CTAs, active states, highlights.
- Avoid adding unrelated accent hues as primary brand colors.
- For technical diagrams, use white, slate, and blue before introducing status colors.

## Typography

The current SVG assets use system-safe fonts for portability. Product and marketing surfaces should use a more intentional family.

### Recommended stack

- **Display:** `Space Grotesk`
- **Body:** `Instrument Sans`
- **Mono:** `IBM Plex Mono`

### Fallbacks

- Display: `"Space Grotesk", "Segoe UI", sans-serif`
- Body: `"Instrument Sans", "Helvetica Neue", Arial, sans-serif`
- Mono: `"IBM Plex Mono", "SFMono-Regular", Consolas, monospace`

### Type scale

| Use | Size | Weight | Notes |
|---|---|---|---|
| Hero title | `56-72px` | `700` | Tight line height, short lines |
| Page title | `40-48px` | `700` | Product and landing pages |
| Section title | `28-32px` | `600` | Use with high contrast |
| Body | `16-18px` | `400-500` | Keep readable and direct |
| Small label | `12-14px` | `500-600` | Meta text, chips, overlines |
| Code | `13-15px` | `400-500` | MCP config, API examples |

### Typography rules

- Headlines should be short, concrete, and operator-focused.
- Use title case for hero and section headings.
- Avoid oversized paragraphs and center-aligned body copy blocks.
- Use mono for commands, keys, JSON, and tool names only.

## Graphic Language

### Visual motifs

- Compass / navigation geometry
- Subtle technical grid
- Directional arrows
- Diagram lines and nodes
- Dark-to-electric gradient fields

### Imagery rules

- Prefer product screenshots, workflow diagrams, and structured UI compositions.
- Use screenshots inside dark frames with soft shadows and blue highlights.
- Avoid stock photos of robots, glowing brains, or generic team meetings.
- If using people, show operators at work rather than lifestyle imagery.

### Iconography

- Use clean geometric icons with medium stroke weight.
- Favor utility metaphors: tools, nodes, pages, layers, routes, settings.
- Keep icon corners slightly rounded, not playful.

## UI Direction

### Surface behavior

- Dark hero sections, light documentation surfaces, and high-contrast CTAs
- Generous spacing
- Rounded corners in the `12-18px` range
- Soft shadows, not heavy glassmorphism
- Motion should communicate flow, not decoration

### Motion guidelines

- Use subtle fades, slide-ins, and staggered reveals
- Use directional motion that supports the "control layer" metaphor
- Keep transitions fast: `160-240ms`
- Avoid bouncy or playful easing

## Voice And Tone

### Brand voice

- Technical
- Direct
- Credible
- Operator-first
- Calm under complexity

### Tone rules

- Explain exactly what the product does
- Prefer specificity over hype
- Lead with capability, then safety
- Use plain language even in technical contexts
- Write as if the reader will evaluate the product critically

### Good copy examples

- "Turn your WordPress site into an MCP server."
- "Give AI agents structured access to content, design, and commerce workflows."
- "Use role-scoped keys to limit what each assistant can do."
- "Build, edit, and automate WordPress operations through real tools."

### Copy to avoid

- "Revolutionary AI magic for websites"
- "One prompt does everything"
- "Unlimited autonomous publishing"
- "Instant enterprise transformation"

## Social And Community Style

- Keep posts short and concrete
- Show prompts, outputs, and before/after screenshots
- Lead with the workflow, not the abstraction
- Mention supported systems only when verified
- Prefer "here is the exact call" over broad claims

## Do / Do Not

### Do

- Use dark surfaces with electric blue emphasis
- Show the product in action
- Keep copy grounded in workflows and permissions
- Use the compass mark consistently
- Preserve strong contrast in every marketing surface

### Do not

- Drift toward generic AI purple branding
- Mix in random neon accents
- Describe the product as a chatbot
- Use novelty visuals that weaken trust
- Publish copy that overstates autonomy or tool coverage
