# MCPWP SEO Competitive Map

Research date: 2026-06-05

## Positioning Thesis

MCPWP should own the category phrase "WordPress MCP server" while differentiating on operator-safe WordPress workflows, Elementor-native implementation, and direct site control from Claude, Cursor, Codex, ChatGPT, and other MCP clients.

The launch site should avoid brittle counts such as "239 tools" and instead use durable claims:

- "WordPress becomes an MCP server"
- "AI clients can safely operate real WordPress workflows"
- "Elementor, content, SEO, media, menus, and site operations"
- "Scoped API keys, auditability, approval-aware workflows"

## Risk Notes

- Do not claim a static tool count in search snippets. The number changes.
- Do not overpromise fully autonomous live publishing. The safer claim is approval-aware site operations.
- Do not lead with "AI website builder"; it puts MCPWP against Elementor/SeedProd broadly. Lead with "WordPress MCP server" and "AI site operations".
- SEO-tooling claims should be gated until `wp_validate_seo_readiness` and `wp_seo_audit_site` are fixed on live.
